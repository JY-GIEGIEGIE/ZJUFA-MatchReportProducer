
### 🎨 全局视觉与色彩规范 (Global Design & Colors)

*   **全局背景**：纯白色 (`#FFFFFF`)。
*   **主色调 (Theme Blue)**：贯穿全局的天蓝色，用于文字、线条、背景色块。建议色值为 `#3498db` 或 Tailwind 的 `blue-500`。
*   **辅助功能色**：
    *   换上绿 (Sub-Up Green)：`#2ecc71`
    *   换下红 (Sub-Down Red)：`#e74c3c`
    *   警告黄 (Warning Yellow)：`#f1c40f`
*   **字体排版**：全局使用无衬线字体（如 `PingFang SC`, `Microsoft YaHei`）。数字部分（时间、号码、比分）建议使用等宽字体或 `DIN Alternate` 以保证视觉对齐。
*   **素材格式**：`CSS (全局变量)`

---

### 1. 头部信息区 (Header Section)

*   **1.1 赛事基础信息**
    *   **设计描述**：分为两行居中显示。第一行为“日期与场地”（如 `260531, 银泉西（北）足球场`），第二行为“赛事名称”。字体颜色为主色调，字号适中，行高宽松。
    *   **素材格式**：`CSS (Text, text-align: center, color: theme-blue)`
*   **1.2 核心比分板**
    *   **设计描述**：居中显示。格式为 `[主队名称] [主队得分]-[客队得分] [客队名称]`。如果是点球大战，在常规比分后方用括号附加点球比分，如 `1-1 (3-1)`。比分数字需加粗 (`font-weight: bold`)，两侧留出适当间距。
    *   **素材格式**：`CSS (Text, Flexbox)`

### 2. 核心时间轴区 (Timeline Section) - *核心组件*

*   **2.1 时间轴主干 (The Axis)**
    *   **设计描述（**：一条居中的垂直细实线，颜色为主色调。
    *   **素材格式**：`CSS (border-left: 2px solid theme-blue)`
*   **2.2 开/终场哨图标 (Start/End Icons)**
    *   **预期效果**：改为极简的扁平化、纯单色（主色调）SVG 图标。顶部使用倾斜的哨子剪影，底部使用秒表剪影。
    *   **素材格式**：`SVG (内联代码，fill: theme-blue)`
*   **2.3 时间节点 (Time Nodes)**
    *   **设计描述**：垂直线上的实心圆圈。背景为主色调，内部文字为纯白色。常规时间显示数字（如 `5`）, 补时阶段用常规时间+补时的时间（`60+1`），点球大战阶段显示汉字“点”。
    *   **素材格式**：`CSS (width/height 固定, border-radius: 50%, background: theme-blue, color: white, flex 居中)`
*   **2.4 连接短线 (Connection Lines)**
    *   **设计描述**：连接时间节点与事件卡片的水平短实线，颜色为主色调。
    *   **素材格式**：`CSS (width 固定, border-top: 2px solid theme-blue)`

### 3. 事件卡片区 (Event Cards) - *需高度结构化*

**通用卡片容器规范**：
*   **设计描述**：主色调背景的圆角矩形（`border-radius: 4px` 或 `6px`），内部文字和图标均为纯白色。
*   **排版方向（极重要）**：
    *   **左侧主队卡片**：内容靠右对齐。DOM 顺序为 `[姓名] [号码#] [图标]`。
    *   **右侧客队卡片**：内容靠左对齐。DOM 顺序为 `[图标] [号码#] [姓名]`。
*   **素材格式**：`CSS (Flexbox, background: theme-blue, color: white)`

**具体事件类型拆解**：

*   **3.1 进球事件 (Goal)**
    *   **设计描述**：单行显示。包含姓名、号码和一个纯白色的足球图标。
    *   **素材格式**：`SVG (纯白色扁平化足球图标)`
*   **3.2 换人事件 (Substitution)**
    *   **预期效果**：保持上下两行结构，上行为换上（绿色向上箭头 + 号码 + 姓名），下行为换下（红色向下箭头 + 号码 + 姓名）。箭头改用现代简洁的无柄箭头或极简实心箭头。
    *   **素材格式**：`SVG (向上箭头 fill: #2ecc71, 向下箭头 fill: #e74c3c)` + `CSS (Flex-direction: column)`
*   **3.3 罚牌事件 (Cards - Yellow/Red)**
    *   **设计描述（忠实还原）**：分上下两行。上行为罚牌图标 + 号码 + 姓名；下行为极小字号的犯规原因（如 `C6 非体育行为`）。罚牌图标为倾斜的圆角矩形，内部带一个感叹号 `!`。
    *   **素材格式**：`SVG (黄牌 fill: #f1c40f, 红牌 fill: #e74c3c, 内部感叹号 fill: white)` + `CSS (下行文字 font-size: 10px 或 12px)`
*   **3.4 点球大战事件 (Penalty Shootout)**
    *   **设计描述**：单行显示。包含点球图标 + 号码 + 姓名。
    *   **图标细节**：罚进为黄色足球图标；罚丢为黄色足球图标上方叠加一个红色的叉号（`X`）。
    *   **素材格式**：`SVG (罚进：黄色足球；罚丢：黄色足球 + 红色交叉线)`

### 4. 底部信息区 (Footer Section)

*   **4.1 裁判名单网格 (Referees Grid)**
    *   **预期效果**：保持 2x2 居中网格布局，左侧图标，右侧姓名。图标改为通用的极简设计：
        *   主裁：简约的哨子或裁判半身剪影。
        *   第一助理裁判：两面交叉的越位旗。
        *   第二助理裁判：单面越位旗。
        *   第四官员：战术板或记事本+笔。
    *   **素材格式**：`SVG (4款极简图标，fill: theme-blue)` + `CSS (Grid 或 Flexbox)`
*   **4.2 图例说明 (Legend)**
    *   **设计描述**：底部最后一行，水平居中排列 4 个图例。每个图例由“彩色足球图标 + 对应颜色的文字”组成。
        *   ⚽ 进球（主色调蓝）
        *   ⚽ 乌龙（红色 `#e74c3c`）
        *   ⚽ 点球（黄色 `#f1c40f`）
        *   ⚽ 点球未进（黄色球 + 红色叉号，文字为黄色）
    *   **素材格式**：`SVG (复用前文的足球图标，仅通过 CSS 改变 fill 颜色)` + `CSS (Flexbox 横向排列, gap 间距)`
