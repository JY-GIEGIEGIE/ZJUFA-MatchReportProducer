# 总体目标
为浙江大学足球协会提供自动化生成赛事战报的浏览器工具。通过结构化的比赛基础信息与赛果输入，组合视觉素材，替换文字与数字产生图文战报，无需手动PS。

# 实现规划
## 技术选型
- 预期为Serverless的单页应用。
- 采用稳定版本的Vue3编写，配合TailwindCSS快速构建UI模板。
- 完成开发后，目标将编译好的静态网页免费托管在GitHub Pages上。
- 引入目前渲染效果更好、对现代 CSS 支持更完善的 dom-to-image-more 作为html转图片的功能库。
## 布局设计与功能对应
- **左侧**：结构化的表单输入区（录入比赛信息、进球、换人等）。
- **右侧**：实时预览区（根据输入实时渲染出的战报 HTML 样式）。宽度暂定1080px与微信公众号图片保持一致。
- **索引栏**：左上点击，点击后展开列表，列表包括所有已经导出或者暂存状态的战报，索引为“赛事名称”与“对阵双方”的字段组合，如这两个信息也没有填完，则只有最后一次编辑的时间戳作为标识（有索引的也显示时间）。
- **导出功能**：点击“生成战报”按钮，直接在浏览器端将 HTML 转换为图片并下载到本地。支持批量导出。
- **暂存功能**：点击“暂存，编辑下一场”按钮，已有的内容（包括填写进度与JSON表单）保存，输入与预览区刷新
>按钮均在页面下方。
### 关于结构化输入的UI
为防止非技术同学填写混乱导致排版崩溃，输入端做如下设计：
1. 基础信息（固定字段）：
日期与场地 （日历选择日期，下拉框选择场地）
赛事名称（下拉框固定字典，若暂无所需名称，支持输入）
对阵双方（同行的两个输入框）
比分（输入框，限自然数）
2. 动态列表（进球、换人、红黄牌等）：
不要给一个大文本框让他们自己敲格式，设计举例（非全部，参考实现）：
给一个 “+ 添加进球” 的按钮。点击后出现一行表单：[进球球员<姓名,号码>] [进球时间(下拉/数字)] [进球类型(常规/点球/乌龙)]。
给一个 “+ 添加换人” 的按钮：[换上球员<姓名，号码>] [换下球员<姓名，号码>] [时间]。
前端代码通过遍历这些数组（如 v-for 或 .map），自动生成排版整齐的 JSON 表单。
3. 首发/替补名单：
可以提供一个多行文本框，提示用户“每行输入一个名字，空格隔开号码”，然后你用 JS 去解析（split('\n')），这样既保证了录入效率，又控制了输出格式。
选做，一期可以不做。
